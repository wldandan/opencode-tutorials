# TalkPro MVP 范围定义

## 产品定位

**TalkPro - 工程师的 AI 职业教练**
**目标用户**: 后端/全栈工程师
**核心价值**: 通过 AI Agent 模拟真实面试，帮助工程师提升能力

---

## MVP 核心理念

**先验证核心价值，再完善辅助功能**

MVP 只包含最核心的训练功能，无需用户注册，让用户能快速体验 AI 面试的价值。

---

## MVP 功能范围

### ✅ 第一版本必须包含

#### 1. 算法面试训练（核心功能）
- 用户选择难度（简单/中等/困难）
- AI 出题（LeetCode 风格）
- 用户提交解答（文字或代码）
- AI 实时追问（流式响应）
- 生成评估报告

#### 2. 系统设计训练（核心功能）
- 用户选择场景（10+ 经典场景）
- AI 引导式讨论
- 用户描述设计方案
- AI 深度挑战
- 生成评估报告

#### 3. 基础 Web UI
- 面试场景页面
  - 对话面板（显示题目和对话历史）
  - 输入框（支持文字和代码）
  - 流式显示 AI 回复
- 简洁的首页（选择训练类型）

#### 4. AI 对话引擎
- 接入 Claude Sonnet 4.5
- 流式响应
- 上下文管理
- 角色扮演（面试官、架构师）

#### 5. 评估系统
- 算法面试评估（5 个维度）
- 系统设计评估（6 个维度）
- 生成反馈报告
- 展示评分和建议

#### 6. 数据持久化
- SQLite 存储对话历史
- 会话管理
- 题库/场景库（JSON 文件或数据库）

---

### ❌ 第一版本不包含

#### 暂缓的功能（后续版本）

**用户相关：**
- ❌ 用户注册/登录
- ❌ 用户档案管理
- ❌ 简历上传和解析
- ❌ 目标岗位 JD 管理

**成长管理：**
- ❌ 训练历史记录（只保存本次会话）
- ❌ 能力雷达图
- ❌ 成长轨迹
- ❌ 薄弱点分析
- ❌ 智能推荐

**个性化：**
- ❌ 根据简历定制题目
- ❌ 难度自适应（用户手动选择）

**其他：**
- ❌ 代码执行沙箱
- ❌ 职场场景训练（晋升答辩等）
- ❌ 多 LLM 支持

## MVP 技术架构（简化版）

### 后端
- **Python 3.10+**
- **FastAPI**（轻量 Web 框架）
- **SQLite**（本地文件数据库，无需安装）
- **Claude API**（Anthropic）
- **WebSocket**（流式响应）

### 前端
- **React 18 + TypeScript**
- **Vite**（快速构建）
- **Tailwind CSS**（样式）
- **Monaco Editor**（代码编辑，可选）
- **WebSocket Client**（实时对话）

### 部署
- **本地运行**（`python run.py` + `npm run dev`）
- **无需云部署**（MVP 阶段）

## MVP 数据模型（简化版）

### Question 表（算法题）
```python
id: UUID
title: String
content: Text
difficulty: Enum (easy, medium, hard)
examples: JSON
solution: Text
```

### Scenario 表（系统设计场景）
```python
id: UUID
title: String
description: Text
requirements: Text
constraints: Text
```

### Session 表（会话）
```python
id: UUID
type: Enum (algorithm, system_design)
question_id: UUID (算法题) 或 scenario_id (系统设计)
messages: JSON (对话历史)
score: JSON (评分)
feedback: Text (反馈)
status: Enum (in_progress, completed)
created_at: DateTime
```

## MVP 用户流程

### 算法面试流程
```
1. 打开网站
2. 点击"算法面试"
3. 选择难度（简单/中等/困难）
4. AI 出题
5. 用户提交解答
6. AI 追问（1-3 轮）
7. 生成评估报告
8. 结束（无需保存，关闭即丢失）
```

### 系统设计流程
```
1. 打开网站
2. 点击"系统设计"
3. 选择场景（微博 Feed、秒杀系统...）
4. AI 引导讨论
5. 用户描述方案
6. AI 挑战（3-5 轮）
7. 生成评估报告
8. 结束（无需保存，关闭即丢失）
```

## MVP 成功指标

### 验证核心假设
- 用户是否愿意使用 AI 面试？
- AI 面试是否比刷题更有价值？
- 用户是否愿意继续使用？

### 目标数据
- 用户完成率 > 50%（开始 → 完成评估）
- 用户满意度 > 3.5/5.0
- 愿意推荐给朋友 > 30%
- 日活用户 > 50

## MVP 风险与应对

### 风险 1：AI 回答质量不稳定
**应对**：
- 优化 System Prompt
- 添加 Few-shot Examples
- 人工审查和调整

### 风险 2：用户不愿意使用
**应对**：
- 提供预设题目快速体验
- 简化操作流程
- 收集用户反馈快速迭代

### 风险 3：成本过高（Claude API）
**应对**：
- 设置每日使用次数限制
- 缓存常见问题
- 后续切换到更便宜的模型
