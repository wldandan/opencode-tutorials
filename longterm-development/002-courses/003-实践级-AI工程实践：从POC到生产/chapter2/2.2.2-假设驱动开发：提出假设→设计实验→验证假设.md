# 2.2.2 假设驱动开发：提出假设→设计实验→验证假设

> **所属章节**：第2章 需求分析与价值验证
> **阅读时长**：10分钟
> **难度等级**：⭐⭐⭐

---

## 引言

很多LLM应用项目的失败，不是因为技术不行，而是因为**假设错了**。

"用户肯定需要这个功能"
"这个技术方案一定能达到90%准确率"
"上线后用户满意度一定很高"

这些假设如果错了，项目就失败了。**假设驱动开发**是一种科学的方法，帮助你系统地提出假设、设计实验、验证假设，避免盲目投入。

---

## 什么是假设驱动开发？

### 核心思想

```
┌─────────────────────────────────────────────────────────────────┐
│                    假设驱动开发循环                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   提出假设 → 设计实验 → 收集数据 → 验证假设 → 做出决策          │
│     ↑                                              ↓          │
│     └────────────── 新的假设 ← 学习调整 ─────────────────────┘ │
│                                                                 │
│   不是"我觉得"，而是"数据证明"                                   │
│   不是"盲目投入"，而是"科学验证"                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 与传统开发的区别

| 维度 | 传统开发 | 假设驱动开发 |
|-----|---------|-------------|
| **起点** | 需求文档 | 假设 |
| **过程** | 按计划开发 | 设计实验验证 |
| **决策** | 主观判断 | 数据驱动 |
| **风险** | 风险后期暴露 | 风险早期发现 |

## 步骤1：提出假设

### 好假设的特点

```
SMART原则：
- Specific（具体的）：明确具体
- Measurable（可衡量的）：可量化
- Testable（可测试的）：可验证
- Ambitious（有野心的）：有价值
- Realistic（现实的）：可达成
```

### 假设的类型

**类型1：价值假设**

```
核心问题：有人愿意用吗？

示例：
❌ 差："用户会喜欢这个功能"
✅ 好："至少60%的受访用户表示愿意使用"

可测试：可以做用户调研验证
```

**类型2：用户假设**

```
核心问题：目标用户是谁？

示例：
❌ 差："我们的用户是所有员工"
✅ 好："客服团队中有60%愿意使用AI工具"

可测试：可以调研验证
```

**类型3：效果假设**

```
核心问题：AI能解决问题吗？

示例：
❌ 差："AI能达到很高的准确率"
✅ 好："AI在简单问题上准确率≥70%"

可测试：可以做POC验证
```

**类型4：成本假设**

```
核心问题：成本可接受吗？

示例：
❌ 差："成本会比较低"
✅ 好："AI处理成本≤人工成本的30%"

可测试：可以成本测算
```

### 企业案例：智能客服的假设

**项目假设列表**：

```
【价值假设】
H1: 企业愿意为智能客服投入（预算≥100万/年）
H2: 智能客服能降低人力成本≥40%

【用户假设】
H3: 至少50%的简单问题用户愿意让AI处理
H4: 用户满意度≥70%

【效果假设】
H5: 简单问题AI准确率≥75%
H6: 复杂问题AI准确率≥50%

【成本假设】
H7: AI单次处理成本≤人工的30%
```

## 步骤2：设计实验

### 实验设计原则

**最小可行实验**：

```
原则：用最小成本验证假设

示例：验证"用户愿意用AI客服"

❌ 大规模实验：
- 开发完整系统
- 上线给所有用户
- 成本：高，周期：长

✅ 最小实验：
- 做一个交互原型
- 找20个用户测试
- 成本：低，周期：短
```

### 实验类型

**类型1：用户访谈实验**

```
目的：验证价值假设和用户假设

方法：
1. 设计访谈问题
2. 找20-30个目标用户
3. 深度访谈

验证：
- H1: 愿意投入？问预算、优先级
- H3: 愿意使用？问使用意愿、顾虑

时间：2-3周
成本：低
```

**类型2：POC实验**

```
目的：验证效果假设

方法：
1. 准备测试数据（500-1000条）
2. 快速开发POC（使用商用API）
3. 内部测试评估

验证：
- H5: 简单问题准确率≥75%？
- H6: 复杂问题准确率≥50%？

时间：4-6周
成本：中
```

**类型3：A/B测试**

```
目的：对比不同方案的效果

方法：
1. 设计两个方案（方案A、方案B）
2. 分流测试（各50%用户）
3. 对比效果指标

验证：
- 哪个方案准确率更高？
- 哪个方案用户更满意？

时间：4-8周
成本：中
```

**类型4：灰度实验**

```
目的：在真实环境中验证

方法：
1. 小范围上线（5-10%用户）
2. 收集真实使用数据
3. 分析评估

验证：
- 真实场景下的效果
- 用户真实反馈

时间：4-12周
成本：高
```

## 步骤3：验证假设

### 数据分析方法

**定量分析**：

```
核心问题：是否达到成功标准？

示例：验证"用户愿意使用AI客服"

数据：
- 调研100人
- 表示愿意使用：65人

分析：
- 样本量：100人，置信度95%
- 意愿率：65% ± 9.5%（置信区间）
- 结论：显著大于50%的目标
- 决策：✅ 假设成立
```

**定性分析**：

```
核心问题：为什么愿意/不愿意？

示例：用户访谈反馈

正面反馈（65%）：
- "24小时服务很方便"
- "响应很快"
- "简单问题解决得好"

负面反馈（35%）：
- "复杂问题不行"
- "有时候答非所问"
- "还是喜欢人工"

分析：
- 简单问题接受度高
- 需要改进复杂问题处理
```

### 假设验证决策矩阵

| 实验结果 | 数据支持 | 决策 |
|---------|---------|------|
| **显著超过目标** | 数据充分 | ✅ 假设成立，加速推进 |
| **刚好达到目标** | 数据基本充分 | ✅ 假设成立，谨慎推进 |
| **接近目标** | 数据部分支持 | ⚠️ 假设部分成立，优化后验证 |
| **远低于目标** | 数据不支持 | ❌ 假设不成立，调整或放弃 |

### 企业案例：假设验证

**假设H5：简单问题AI准确率≥75%**

**POC实验**：
- 测试集：1000个简单问题
- 模型：GPT-4
- 评估方法：专家标注

**结果**：
- 准确率：78%
- 置信区间：±3%
- 结论：显著≥75%

**决策**：✅ 假设成立，可以进入MVP阶段

## 假设驱动开发实战

### 完整流程示例

**项目：智能文档处理**

**阶段1：提出假设**

```
价值假设：
V1: 企业愿意投入（预算≥50万）
V2: 能提升处理效率≥3倍

效果假设：
E1: 表格识别准确率≥90%
E2: 字段提取F1≥85%

成本假设：
C1: 单份处理成本≤人工的20%
```

**阶段2：设计实验**

```
实验1：用户访谈（2周）
  - 验证V1、V2
  - 样本：20个潜在客户

实验2：POC开发（6周）
  - 验证E1、E2
  - 数据：100份文档

实验3：成本测算（1周）
  - 验证C1
  - 方法：成本模型计算
```

**阶段3：验证假设**

```
实验1结果：
  - V1: 80%客户愿意投入 ✅
  - V2: 期望提升5倍 ✅
  → 价值假设成立

实验2结果：
  - E1: 表格识别准确率92% ✅
  - E2: 字段提取F1 78% ⚠️
  → 效果假设部分成立

实验3结果：
  - C1: 单份成本为人工的15% ✅
  → 成本假设成立
```

**阶段4：决策**

```
综合评估：
- 价值假设成立 ✅
- 效果假设部分成立 ⚠️
- 成本假设成立 ✅

决策：
✅ 进入MVP阶段
优化方向：
- 重点优化字段提取
- 考虑人工审核兜底
```

## 总结

### 假设驱动开发回顾

```
提出假设 → 设计实验 → 验证假设 → 做出决策
   ↑                              ↓
   └────────── 学习调整 ←────────────┘
```

### 关键要点

```
✅ 假设要SMART：具体、可衡量、可测试
✅ 实验要最小：用最小成本验证
✅ 数据要客观：定量+定性结合
✅ 决策要科学：基于数据，不凭感觉
```

### 常见错误

```
❌ 假设模糊，无法验证
❌ 实验太大，成本太高
❌ 只看定量，忽视定性
❌ 数据解读有偏差
❌ 忽视实验的局限性
```
