## 案例背景

### 企业信息

```
企业名称：某中型互联网公司
业务类型：在线教育
规模：500人团队，日活用户10万
痛点：客服人力成本高，用户增长放缓
```

### 项目背景

```
现状：
- 客服团队30人
- 日均处理咨询3000次
- 用户满意度65分
- 年人力成本：450万

问题：
- 高峰期响应慢（平均5分钟）
- 简单问题占用大量人力
- 用户满意度下降
- 成本压力大

目标：
- 用AI客服提升效率
- 降低人力成本
- 提升用户满意度
```

## 实战练习：需求分析

### 练习1：需求收集（三步法）

#### 步骤1：访谈

**三类人的访谈结果摘要**：

| 角色 | 核心观点 |
|-----|---------|
| **客服主管** | "每天3000次咨询，70%是简单问题，太浪费人力了" |
| **客服代表** | "简单问题占用了70%时间，复杂问题没时间处理" |
| **产品负责人** | "用户抱怨响应慢，满意度下降，影响口碑" |
| **技术负责人** | "有知识库，但搜索不好用，客服主要靠个人笔记" |

#### 步骤2：观察

**跟岗观察发现**：

```
观察对象：5个客服代表，各观察半天

发现1：高频问题集中
  - 查课程进度：35%
  - 退课/换课：25%
  - 技术问题：20%
  - 其他：20%

发现2：知识库使用率低
  - 系统有知识库，但只有30%查询
  - 70%靠问同事或个人笔记

发现3：复杂问题处理流程复杂
  - 需要查询多个系统
  - 经常跨部门协作
  - 平均处理时间15分钟
```

#### 步骤3：数据

**数据分析结果**：

```
数据1：咨询分布
  - 日均3000次咨询
  - 高峰期（10-16点）占70%
  - 简单问题（查询类）占75%
  - 复杂问题（操作类）占25%

数据2：处理时长
  - 平均处理时长：5分钟
  - 简单问题：2分钟
  - 复杂问题：15分钟

数据3：用户满意度
  - 当前满意度：65分
  - 响应时间满意度：50分（主要痛点）
  - 问题解决满意度：75分

数据4：知识库
  - 知识库条目：5000条
  - 覆盖率：80%
  - 但搜索准确率只有40%
```

### 练习3：技术需求转换

**业务目标 → 技术指标 → 验收标准**

| 业务目标 | 技术指标 | 验收标准 |
|---------|---------|---------|
| **降低人力成本50%** | AI解决率≥70% | 测试集1000条，准确率≥70% |
| **提升24小时服务** | 系统可用性≥99.5% | 7×24监控，月停机<3.6小时 |
| **提升满意度到75分** | 满意度≥75% | 用户调研，满意度≥75分 |
| **处理简单问题** | 响应时间≤1分钟 | 95%查询≤1分钟 |

**分场景标准**：

| 场景 | 占比 | 准确率目标 | 响应时间 |
|-----|------|-----------|---------|
| 查课程进度 | 35% | ≥95% | ≤30秒 |
| 退课/换课 | 25% | ≥90% | ≤2分钟 |
| 技术问题 | 20% | ≥80% | ≤3分钟 |
| 其他 | 20% | ≥70% | ≤2分钟 |

### 练习4：MVP设计

**核心假设**：

```
H1: 用户愿意使用AI客服（意愿率≥60%）
H2: AI能准确处理简单问题（准确率≥75%）
H3: AI能降低人力成本≥40%
H4: 用户满意度能提升到≥70分
```

**MVP范围**：

```
功能范围：
✅ 只处理3类简单问题（查询、退课、技术）
✅ 只做文字交互
✅ 只服务100个用户（试点）
✅ 只运行1个月

不做：
❌ 复杂问题（转人工）
❌ 语音交互
❌ 全量用户
❌ 长期运营
```

**验收标准**：

```
指标1：用户意愿率
- 测试：50个用户调研
- 目标：≥60%

指标2：准确率
- 测试：500个测试问题
- 目标：≥75%

指标3：成本
- 测试：对比AI前后人力
- 目标：≥40%节省
```

### 练习6：价值树分析

**直接价值**：

| 价值类型 | 计算过程 | 价值 |
|---------|---------|------|
| **人力节省** | 30人×10万×50% = 150万 | 150万/年 |
| **时间节省** | 2分/次×3000次/天×250天×30元/时 = 22.5万 | 22.5万/年 |
| **收入增长** | 500万收入×5%转化率提升 = 25万 | 25万/年 |
| **合计** | | **197.5万/年** |

**间接价值**：

| 价值类型 | 估算 | 价值 |
|---------|------|------|
| **员工满意度** | 满意度+10分，离职率-3% | 30万/年 |
| **用户体验** | 响应时间缩短，满意度+10分 | 50万/年 |

**战略价值**：

| 价值类型 | 估值 |
|---------|------|
| **AI能力建设** | 300万 |
| **数据资产积累** | 200万 |
| **创新文化** | 100万 |
| **合计** | **600万** |

**总价值**：

```
总价值 = 直接 + 间接 + 战略
      = 197.5 + 80 + 600
      = 877.5万/年
```

## 练习答案

### 需求收集结果

```
核心发现：
1. 70%是简单问题，可以自动化
2. 知识库有但搜索不好用（40%准确率）
3. 高峰期压力大，响应慢是最大痛点
4. 复杂问题需要专业知识，AI难以处理

核心需求：
1. AI处理简单查询（效率）
2. 提升24小时响应能力（体验）
3. 优化知识库搜索（效率+体验）
```

### 需求分析结论

```
优先级排序：
P1（立即做）：
- AI处理简单查询（500万价值，中难度）
- 提升24小时响应（体验价值，低难度）

P2（规划做）：
- 智能推荐课程（200万价值，高难度）
- 提升知识库准确率（效率+体验，中难度）

P3（考虑做）：
- AI辅助复杂问题（100万价值，高难度）
```

### MVP设计总结

```
MVP范围：
- 只做3类简单问题
- 只服务100个用户
- 只运行1个月

核心目标：
- 验证用户接受度（≥60%）
- 验证AI可行性（≥75%准确率）
- 验证成本节省（≥40%）

预期结果：
- 如果假设成立，进入完整开发
- 如果不成立，调整方向或放弃
```

## 实战练习总结

### 完整流程回顾

```
1. 需求收集（三步法）
   - 访谈三类人
   - 跟岗观察
   - 数据分析

2. 需求分析（四象限）
   - 分类：效率/体验/风险/收入
   - 评估：价值/难度
   - 排序：P1-P4

3. 技术转换
   - 业务目标 → 技术指标
   - 分场景设定标准

4. MVP设计
   - 核心假设定义
   - MVP范围界定
   - 验收标准设定

5. 用户故事地图
   - 用户旅程定义
   - 功能卡片设计
   - 优先级排序

6. 价值树分析
   - 直接价值计算
   - 间接价值估算
   - 战略价值描述
```
